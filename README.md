OTT DIET
구독 서비스 소비 합리화를 위한 가성비 분석 안드로이드 애플리케이션

1. 프로젝트 개요
넷플릭스나 유튜브 프리미엄 같은 구독 서비스를 여러 개 쓰다 보면, 매달 돈은 나가는데 막상 안 쓰는 앱들이 생기기 마련입니다.
이 앱은 단순히 매달 나가는 총액만 보여주는 기존 가계부와 다릅니다.
사용자가 '한 달에 실제 몇 번이나 접속하는지'를 입력받아 '1회 접속당 진짜 지불하고 있는 비용'을 계산해 줍니다.
돈만 내고 거의 안 쓰는 '저효율 서비스'를 한눈에 파악하고 과감히 해지(다이어트)할 수 있도록 돕는 실용적인 지출 관리 솔루션입니다.

2. 개발 환경
Platform: Android
Language: Java
Target SDK: 34

3. 주요 기능
로컬 데이터 저장 (SharedPreferences): 사용자가 입력한 구독료를 디바이스에 저장해 둡니다.
앱을 완전히 껐다 켜도 기존 데이터가 자동으로 복원되어 매번 다시 입력할 필요가 없습니다.

동적 UI 구성: 메인 화면에서 선택한 서비스 개수만큼만 다음 화면에서 주간 접속 일수를 입력하는 슬라이더(SeekBar)가 실시간으로 생성(Inflation)됩니다.

가성비 분석 및 정렬: [월 구독료 / (주간 접속일 * 4)] 연산으로 1회당 비용을 뽑아냅니다.
Comparable 인터페이스를 구현하여 가성비가 가장 최악인(돈 낭비가 심한) 서비스가 목록 맨 위에 오도록 정렬합니다.

기준점 기반 해지 제안: 1회 접속 비용이 3,000원을 넘어가면 가성비가 심각하게 무너진 것으로 판단하여 '비효율' 그룹으로 분류하고 해지를 강력 권장합니다.

해지 페이지 바로가기 (Deep Linking): 가장 가성비가 떨어지는 'Worst 1위' 서비스의 공식 구독 해지 웹페이지 링킹을 연결해 둡니다.
Intent.ACTION_VIEW를 통해 앱 내 버튼 클릭 한 번으로 해지 화면까지 곧바로 이동합니다.

분석 리포트 공유: 유지/주의/해지 3단계로 분류된 분석 결과를 Intent.ACTION_SEND를 이용해 텍스트 형태로 카카오톡 등 메신저에 바로 공유할 수 있습니다.
모바일 가독성을 고려해 불필요한 줄바꿈 없이 깔끔하게 한 줄 요약 포맷으로 전달됩니다.

4. 구조 및 동작 방식
SplashActivity: 앱 시작 시 첫인상을 주는 화면입니다.
Handler를 통해 1.2초간 스플래시 화면을 띄운 뒤 메인으로 넘어가며, finish()를 호출해 사용자가 뒤로가기를 눌러도 스플래시 화면이 다시 나오지 않도록 생명주기를 꼬이지 않게 처리했습니다.

MainActivity: 이용 중인 서비스를 체크하고 월 구독료를 적는 화면입니다.
최신 단말기의 하단 제스처 바나 소프트키와 앱 UI가 겹쳐 클릭이 씹히는 문제를 막기 위해 fitsSystemWindows="true" 속성으로 레이아웃을 최적화했습니다.

UsageActivity: 선택한 서비스들의 주간 사용 빈도(0~7일)를 슬라이더로 조절하는 화면입니다.
드래그할 때마다 리스너가 반응해 한 달 환산 횟수 문구가 실시간으로 바뀝니다.

ResultActivity: 최종 분석 리포트 화면입니다.
가성비 순위가 정렬되어 출력되며, 분석 결과에 맞춰 하단 버튼이 동적으로 활성화(가성비가 모두 좋으면 유지 추천 버튼으로 변경)됩니다.
